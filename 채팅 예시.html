1 요구사항 전달

U

일차의료 지원센터의 매뉴얼 등의 자료를 배포하기 위한 홈페이지를 구축하려고 해. 사용대상자는 환자 및 일차의료 관련 종사자를 대상으로 하고 있어. 그리고 모바일로도 가독성이 좋았으면 좋겠어.

업로드 등 관리는 관리자만 로그인해서 할 수 있고, 사용자는 로그인 과정 없이 볼 수 있도록 하고 싶어. 그리고 관리자는 웹에서 공지사항/고객문의 등 새로운 카테고리를 새로 생성할 수 있었으면 좋겠어.

클로드 코드를 사용할 수 있는 환경이야. Next.js / Supabase / GitHub / Vercel 모두 사용가능해.

일차의료지원센터의 매뉴얼을 모아놓은 곳은 노션의 https://www.notion.so/2026-2fc1d3786d3d80f99171d2c1ce96cb3a 이고, 홈페이지 제작 시 https://health.kdca.go.kr/healthinfo/를 reference로 참고해보고 싶어.

너희 의견을 제시해주고, 이제 어떻게 시작해야할지 알려주면 좋겠어.

2 아키텍처 제안 및 개발 계획

C

참고 사이트(국가건강정보포털)를 조사한 결과, 카테고리별 건강정보를 체계적으로 분류해서 제공하는 공공 포털이었습니다. 이를 참고하되 더 현대적이고 심플한 형태로 구현하기로 했습니다.

기술 스택

영역기술
프론트엔드Next.js (App Router) + Tailwind CSS
백엔드/DBSupabase (PostgreSQL + Storage + Auth)
배포Vercel
코드관리GitHub
개발도구Claude Code

핵심 설계 포인트

  1. Supabase RLS — 관리자만 CRUD, 일반 사용자는 읽기만 가능하도록 DB 레벨에서 제어
  2. 동적 카테고리 시스템 — categories 테이블에 type 필드(library/board/inquiry)로 자료실·게시판·문의함 구분
  3. 모바일 우선 설계 — Tailwind 반응형 클래스

주요 페이지 구성

  • 메인 — 센터 소개 + 최신 자료/공지 하이라이트
  • 자료실 — 카테고리별 매뉴얼 목록 + PDF 미리보기/다운로드
  • 공지사항 — 관리자가 생성한 게시판
  • 고객문의 — 문의 폼 (Supabase에 저장)
  • 관리자 대시보드 (/admin) — 로그인 후 자료 업로드, 카테고리 관리, 게시물 CRUD

DB 테이블

categories    (id, name, slug, type, sort_order, created_at)
posts         (id, category_id, title, content, attachments, created_at)
files         (id, category_id, title, description, file_url, created_at)
inquiries     (id, name, email, content, replied, created_at)

개발 단계 (권장)

  • 1단계 — 프로젝트 초기 세팅 (Next.js + Supabase 패키지 설치)
  • 2단계 — Supabase 프로젝트 생성 (테이블, RLS, Storage, Auth)
  • 3단계 — Claude Code로 핵심 기능 개발 (레이아웃 → 메인 → 자료실 → 관리자 → 카테고리 → 게시판 → 문의)
  • 4단계 — GitHub → Vercel 자동 배포

노션 자료 마이그레이션 — 양이 많지 않다면 관리자 대시보드 완성 후 직접 업로드하는 것이 더 빠름.

4 와이어프레임 3종 제작

C

3종의 와이어프레임을 인라인 위젯으로 제작했습니다.

① 데스크톱 메인 페이지
상단 네비게이션(로고 + 메뉴 + 관리자) → 히어로(제목 + 검색바) → 카테고리 4열 그리드
→ 최신 매뉴얼 3열 카드 → 공지사항 리스트 → 푸터
② 모바일 메인 + 관리자 대시보드
모바일: 햄버거 메뉴, 2열 카테고리, 하단 탭바(홈·자료실·공지·문의)
관리자: 좌측 사이드바(6개 메뉴), 통계 카드 3개, 자료 관리 테이블
③ 자료실 + 카테고리 관리
자료실: 좌측 카테고리 사이드바 + 파일 리스트(아이콘+제목+크기+다운로드)
카테고리 관리: 드래그 정렬 목록 + 새 카테고리 추가 폼(이름·유형·설명)

디자인 방향 요약

  • 컬러: 초록(#1D9E75) 메인 — 의료/건강 분야에 적합
  • 레이아웃: 데스크톱은 사이드바 + 콘텐츠, 모바일은 하단 탭 네비게이션
  • 관리자: 좌측 메뉴 대시보드, 카테고리를 3가지 유형(자료실·게시판·문의함)으로 동적 생성
  • 카테고리 관리: 드래그 순서 변경, 유형 선택 방식

7 Claude Code 파일 업로드 위치

U

클로드 코드는 클라우드를 사용하고 있는데, md 파일은 어디로 업로드해야하지?

C

별도로 업로드할 필요가 없습니다. Claude Code는 현재 작업 디렉토리의 파일을 직접 읽을 수 있으므로, 프로젝트 폴더 루트에 파일을 넣으면 됩니다.

# 1. 프로젝트 폴더 생성
mkdir primary-care-portal
cd primary-care-portal

# 2. 다운로드한 PROJECT_SPEC.md를 이 폴더에 넣기

# 3. Claude Code 실행
claude

# 4. 대화에서 지시
> PROJECT_SPEC.md 파일을 읽고 Phase 1부터 순서대로 프로젝트를 구축해줘

클라우드든 로컬이든 동작 방식은 동일합니다.

8 GitHub 활용 방법

U

깃허브에 md 파일을 올려서 하면 안돼?

C

됩니다! 오히려 그게 더 좋은 방법이에요. GitHub에 레포를 먼저 만들고 PROJECT_SPEC.md를 올려두면, Claude Code에서 바로 작업할 수 있습니다.

# 1. GitHub에서 레포 생성 (예: primary-care-portal)

# 2. 로컬에 클론
git clone https://github.com/[username]/primary-care-portal.git
cd primary-care-portal

# 3. PROJECT_SPEC.md 넣고 푸시
cp ~/Downloads/PROJECT_SPEC.md .
git add PROJECT_SPEC.md
git commit -m "프로젝트 명세서 추가"
git push

# 4. 해당 폴더에서 Claude Code 실행
claude

# 5. 지시
> PROJECT_SPEC.md를 읽고 Phase 1부터 프로젝트를 구축해줘

이렇게 하면 Claude Code가 만드는 코드도 같은 레포에 바로 커밋할 수 있고, 나중에 Vercel 연동도 이 레포에서 바로 하면 되니까 워크플로우가 깔끔합니다.
